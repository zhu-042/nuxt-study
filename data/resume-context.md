## 一、项目背景

项目名称：**万能小in** — AI 内容创作与知识管理平台

技术栈：Nuxt 3 / Vue 3 / TypeScript / TailwindCSS / Pinia / Axios / Tiptap / SSE / WebSocket

项目有两个版本：

- **C 端版本**（master 分支）：面向个人用户，40+ 页面路由
- **B 端团队版**（team-prod 分支）：面向企业团队，基于 C 端改造，新增团队管理后台、权限体系、多租户架构

## 二、用户负责的模块（C 端）

### 模块 1：AI 对话（Chat）

- SSE 流式对话引擎（@microsoft/fetch-event-source）
- WebSocket 实时语音识别（AudioContext + PCM 编码）
- Markdown 渲染管线（KaTeX + highlight.js + citation 引用标记 + 表格导出 Excel）
- SHA256 文件秒传
- 智能滚动控制（节流 + 底部阈值判断）
- 多场景复用（普通聊天 / 知识库文件夹 / 单文件）

### 模块 2：知识库（Library）

- Splitpanes + Teleport + CSS Container Queries 响应式分栏布局
- 六步文件上传流程（SHA256 → 秒传检查 → 预签名 URL → COS 直传 → 后端注册 → 添加到知识库）
- 多格式文件预览（PDF / Office / Markdown / 音频）
- 多功能 Tab 系统（提问 / 思维导图 / 翻译 / 笔记 / PPT），defineAsyncComponent 懒加载
- 翻译与 PDF 联动
- Tiptap 笔记编辑器（自动保存 + 导出 Word）
- 文件状态轮询（增量更新）

### 模块 3：内容资讯与学术搜索（News / Articles / Scholar）

- 游标分页（from_id）替代 offset 分页
- 个性化订阅（知识库推荐 / 自定义标签）
- WordPress REST API 集成
- IndexedDB 缓存管理器（ScholarCacheManager）
- 学术搜索多维度筛选 + 关键词高亮
- 搜索次数消耗管理

### 模块 4：SEO 写作宝典（Manual）

- Nuxt 3 SSR + WordPress CMS 解耦架构
- SSR 内网地址访问 API 降低 TTFB
- 三层 TDK 降级策略
- JSON-LD 结构化数据 + Sitemap + Robots
- 文档付费下载 + AI 在线编辑商业闭环

### 模块 5：B 端团队版改造

- 团队管理后台（概览 / 成员 / 小组 / 智能体 / 权益）
- 三级角色权限体系（创建者 / 管理员 / 普通成员）
- 智能体市场（拖拽排序 + 可见性控制 + 一键上架）
- 基于二级域名的多租户架构（域名识别团队 → 品牌配置 → API 自动注入团队标识）
- Excel 批量导入成员

## 六、关键技术点速查（面试前快速复习用）


| 技术点               | 一句话总结                                                             |
| ----------------- | ----------------------------------------------------------------- |
| SSE               | 基于 HTTP 的服务端单向推送，用 fetch-event-source 支持 POST，按 item_id 结构化管理流式消息 |
| WebSocket         | 全双工通信，语音识别用：浏览器采集音频 → PCM 编码 → WS 发送 → 实时返回识别文字                   |
| Teleport          | 组件在不同 DOM 容器间迁移，实例不销毁状态保留，解决大屏分栏/小屏抽屉切换问题                         |
| Container Queries | 基于容器自身宽度而非视口宽度的响应式，解决 Splitpanes 拖拽场景下媒体查询失效的问题                   |
| SHA256 秒传         | 上传前算哈希问服务端有没有，有就直接复用跳过上传                                          |
| 预签名 URL           | 后端用 OSS 密钥生成带签名的临时上传地址，前端直传 OSS 不经过业务服务器                          |
| markRaw           | 告诉 Vue 不要对组件对象做响应式代理，避免性能浪费                                       |
| 游标分页              | 用 from_id 替代 offset，解决实时更新场景下数据偏移问题                               |
| IndexedDB 缓存      | 学术搜索结果按参数组合缓存，支持单条状态更新和路由离开自动清理                                   |
| 多租户               | 二级域名识别团队 → 加载品牌配置 → API 高阶函数自动注入 teamId                           |
