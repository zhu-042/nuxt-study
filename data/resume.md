### 万能小in — AI 内容创作平台

**技术栈**：Nuxt 3 / Vue 3 / TypeScript / TailwindCSS / Pinia / Axios / Tiptap / SSE / WebSocket

**项目简介**：面向 C 端用户的 AI 内容创作与知识管理平台，集 AI 对话、知识库、学术搜索、内容资讯、SEO 写作宝典等功能于一体。项目覆盖 **60+ 页面路由**、**27 个 Pinia Store**、**16 个 API 模块**，支持 Web、小程序 WebView 等多端适配。同时主导了 **B 端团队 SaaS 版本**的改造，基于二级域名实现多租户隔离，新增团队管理后台、三级权限体系和智能体市场等企业级功能。

#### 一、AI 对话模块

负责多轮 AI 对话系统的整体开发，支持文本、语音、图片、文件等多模态输入方式。

- 基于 **SSE（Server-Sent Events）** 实现 AI 流式对话引擎，使用 fetch-event-source 发起流式请求，按消息 ID 进行结构化管理，支持 Agent 多步推理过程的**时间线可视化展示**（思考 → 工具调用 → 最终回答）
- 基于 **WebSocket + Web Audio API** 实现实时语音识别，通过 AudioContext 采集麦克风音频并编码后流式发送至 ASR 服务，实现边说边转文字，支持句子级增量识别和字数限制自动停止
- 构建完整的 **Markdown 渲染管线**，集成 KaTeX 数学公式、代码高亮、自定义引用标记插件（hover 提示 + 侧边栏联动），支持表格一键导出 Excel（动态加载，按需引入）
- 实现 **SHA256 文件秒传**机制，上传前计算文件哈希并与服务端比对，已存在则直接复用，减少重复上传；支持图片粘贴/拖拽上传、文档上传、知识库文件选择等多种方式
- 设计**智能滚动控制**策略，通过节流监听 + 底部阈值判断，在 AI 流式输出时自动滚动，用户手动上翻时暂停，避免打断阅读体验
- 同一套对话引擎通过场景枚举复用于普通聊天、知识库文件夹聊天、单文件聊天等多个场景

#### 二、知识库模块

负责个人知识库管理系统的开发，支持多格式文件的上传、预览、AI 交互和知识提取。

- 基于 **Splitpanes + Teleport + CSS Container Queries** 实现响应式可拖拽分栏布局，大屏拖拽分栏、小屏自动切换抽屉模式，组件通过 Teleport 在不同 DOM 容器间无缝迁移
- 设计并实现**六步文件上传流程**：SHA256 计算 → 秒传检查 → 获取 COS 预签名 URL → 对象存储直传（带进度条）→ 后端注册 → 添加到知识库，支持 WPS 格式自动转换和空间容量校验
- 实现**多格式文件预览**：PDF 使用自定义 PdfFrame，Office 文件对接微软 Office Online Viewer，Markdown/TXT 自定义渲染，音频文件集成 wavesurfer.js 播放器
- 开发文件详情页**多功能 Tab 系统**（提问 / 思维导图 / 翻译 / 笔记 / PPT），采用异步组件懒加载 + 缓存策略，切换 Tab 不销毁已加载组件，减少首屏体积
- 实现**翻译与 PDF 联动**：点击翻译段落通过 EventBus 高亮 PDF 对应位置，PDF 翻页自动滚动翻译内容
- 基于 **Tiptap** 开发笔记编辑器，支持富文本编辑、从 AI 对话一键插入内容、debounce 自动保存、导出 Word
- 文件上传后通过**定时轮询**批量查询处理状态，采用增量更新策略避免替换整个数组，保持响应式性能

#### 三、内容资讯与学术搜索模块

- 资讯列表采用**游标分页**替代传统 offset 分页，解决实时更新场景下的数据偏移问题，支持按日期切换和无限滚动加载

### EssayPass — AI 论文写作与检测平台（国际版）

**技术栈**：Next.js 15 / React 19 / TypeScript / TailwindCSS / Zustand / Radix UI / next-intl / NextAuth / Sentry

**项目简介**：面向海外市场的 AI 论文写作与检测 SaaS 平台，支持 AI 论文生成、AI 检测、查重等核心功能，集成 WordPress 博客系统与多语言 SEO 体系，采用 Next.js App Router 架构，支持英文/中文国际化。项目覆盖 **46 个页面路由**、**109 个业务组件**，国际化覆盖 **58 个翻译命名空间**，**19 个页面**配置独立 SEO Metadata，**7 个核心页面**注入 JSON-LD 结构化数据。

#### 一、AI 检测与查重模块

负责 AI 内容检测与论文查重功能的完整链路开发，覆盖从文件上传到检测报告展示的全流程。

- 开发**文件上传交互组件**，支持拖拽/点击上传 PDF/DOCX（最大 100MB），集成登录校验与文件缓存机制，上传成功后本地缓存避免重复上传，通过业务类型标识自动创建对应检测任务
- 基于 **OSS 预签名 URL** 实现文件直传，前端通过后端获取临时签名地址后直传对象存储，不经过业务服务器，降低服务端带宽压力
- 开发**任务详情 Dashboard**，服务端并行拉取交付物与文件预览数据实现 SSR，客户端检测到已有数据则跳过重复请求；左侧通过 **iframe 隔离**预览 PDF/DOCX，右侧展示 AI 检测与查重报告卡片（百分比、等级、查看/下载）
- 封装**通用状态机组件**统一管理任务生命周期（进行中 → 完成/失败），通过类型参数区分写作与检测两条产品线，减少重复代码
- 开发 AI 检测与查重**子报告页**，复用写作模块的报告查看器，通过回调动态修正导航路径，确保跨产品线跳转的正确性

#### 二、博客系统与内容管理

负责对接 WordPress CMS 的博客系统开发，实现文章展示、分类浏览、作者页等完整内容体系。

- 封装 **WordPress REST API** 统一请求层，实现文章列表/详情/分类/作者的完整 CRUD，通过嵌入参数一次性获取关联数据（作者、分类、特色图），利用响应头总页数驱动分页
- 实现**中英文内容覆盖**机制，通过 WordPress 自定义 meta 字段在中文环境下自动替换标题与正文，单篇文章支持双语展示
- 开发 **TOC 目录导航**组件，正则解析 H2/H3 标题生成中英文锚点（支持 Unicode），自动去重，配合 **IntersectionObserver** 实现滚动高亮当前章节
- 将 WordPress 正文中的图片标签替换为 **Next.js Image 组件**，优化 LCP/CLS 指标；外链自动添加安全属性（新窗口打开 + nofollow）
- 实现特色图**容错降级**处理，当 API 返回错误对象时自动跳过，避免将错误信息当作图片 URL 渲染
- 文章详情页注入完整的 **JSON-LD 结构化数据**（Article + FAQPage + BreadcrumbList），正文内图片自动提取进 Schema，配合静态生成预渲染最多 100 篇文章页面

#### 三、SEO 营销体系与落地页

负责首页、写作产品页、检测产品页等多个营销落地页的开发，核心目标为 SEO 获客与转化。
