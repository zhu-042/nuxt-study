interface vs type

TypeScript 核心区别 — 交互式代码示例（可直接在浏览器阅读）

1. 声明合并（Declaration Merging） interface 独有
interface — 同名自动合并
interface User {
  name: string;
}

interface User {
  age: number;
}

// 合并结果: { name: string; age: number }
const user: User = { name: "Alice", age: 25 };
type — 同名直接报错
type User = {
  name: string;
}

// Error: Duplicate identifier 'User'
type User = {
  age: number;
}
应用场景：扩展第三方库类型时非常有用，比如给 Window、Express.Request 增加自定义字段。
2. 联合类型 & 判别联合（Discriminated Union） type 独有
type — 联合类型
type Status = "loading" | "success" | "error";

type ID = string | number;
type — 判别联合（状态机建模）
type Action =
  | { type: "add";    payload: Item }
  | { type: "remove"; id: string }
  | { type: "clear" };

function reducer(state: State, action: Action) {
  switch (action.type) {
    case "add":
      // action.payload 自动推断为 Item
      return { ...state, items: [...state.items, action.payload] };
    case "remove":
      // action.id 自动推断为 string
      return { ...state, items: state.items.filter(i => i.id !== action.id) };
  }
}
interface 做不到：interface 无法表达 A | B 联合，只能描述单个对象形状。
3. 元组 & 基本类型别名 type 独有
type — 元组和基本类型别名
// 元组
type Pair = [string, number];
const p: Pair = ["age", 25];

// 基本类型别名
type Milliseconds = number;
type UUID = string;

// React 常见用法：事件处理函数类型
type ClickHandler = (e: MouseEvent) => void;
4. 继承与扩展方式对比 两者都能做
interface — 用 extends
interface Animal {
  name: string;
}

interface Dog extends Animal {
  breed: string;
}

// 多继承也可以
interface Pet extends Animal, Serializable {
  owner: string;
}
type — 用交叉类型 &
type Animal = {
  name: string;
};

type Dog = Animal & {
  breed: string;
};

// 组合多个类型
type Props = BaseProps & ThemeProps & RouterProps;
区别：extends 遇到属性冲突时直接报错（更安全）；& 交叉类型会尝试合并，可能产生 never（更灵活但需注意）。
5. 映射类型 & 条件类型（类型体操） type 独有
type — 映射类型
// 把所有属性变为可选
type MyPartial<T> = {
  [K in keyof T]?: T[K];
};

// 把所有属性变为只读
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};
type — 条件类型
// 提取 Promise 内部类型
type UnwrapPromise<T> =
  T extends Promise<infer U> ? U : T;

// UnwrapPromise<Promise<string>> → string
// UnwrapPromise<number>          → number

// 排除 null 和 undefined
type NonNullable<T> =
  T extends null | undefined ? never : T;
type — 模板字面量类型
type EventName = `on${Capitalize<"click" | "focus" | "blur">}`;
// → "onClick" | "onFocus" | "onBlur"
6. 类实现契约（class implements） 两者都能做
interface 和 type 都可以被 class 实现
interface ILogger {
  log(msg: string): void;
}

type TLogger = {
  log(msg: string): void;
};

// 两种写法都合法
class ConsoleLogger implements ILogger {
  log(msg: string) { console.log(msg); }
}

class FileLogger implements TLogger {
  log(msg: string) { /* 写入文件 */ }
}
但注意：class 不能 implements 联合类型，如 type A = X | Y 是不行的。
7. 属性冲突时的行为差异 关键区别
interface extends — 冲突直接报错（安全）
interface A { x: string; }
interface B { x: number; }

// Error: Type 'number' is not assignable to type 'string'
interface C extends A, B {}
type & — 冲突静默产生 never（危险）
type A = { x: string };
type B = { x: number };

type C = A & B;
// C 的 x 属性变成 string & number → never
// 不会报错，但赋值时任何值都不满足

总结对比表

能力 interface type
声明合并 ✓ ✗
extends 继承 ✓ ✗ (用 & 代替)
联合类型 ✗ ✓
元组 ✗ ✓
映射类型 ✗ ✓
条件类型 ✗ ✓
基本类型别名 ✗ ✓
class implements ✓ ✓ (仅对象类型)
编译性能 ✓ 更优 交叉类型复杂时较慢
面试口诀：描述对象"长什么样"用 interface，做类型运算和组合用 type。真正的区分点只在 声明合并 和 类型运算能力 上。
